## 技术细节

### 遇到的问题及解决方案

#### 问题1: 数据库锁定 (database is locked)
- **现象**: 原始脚本运行时出现数据库锁定错误
- **原因**: 多个数据库连接同时操作，SQLite不支持高并发写入
- **解决方案**:
  1. 使用线程锁 (`threading.Lock`) 保护数据库连接
  2. 增加连接超时时间 (`timeout=30.0`)
  3. 批量插入数据，减少连接次数
  4. 使用 `INSERT OR REPLACE` 避免主键冲突

#### 问题2: 脚本执行缓慢
- **现象**: 处理每只股票需要约3-5秒
- **原因**: 每个股票都需要调用多个Tushare API
- **解决方案**:
  1. 增加API限流控制 (`time.sleep(0.1)`)
  2. 分批处理股票（每批50只）
  3. 批量生成数据后一次性插入

#### 问题3: 表结构不匹配
- **现象**: 测试脚本与实际表结构不一致
- **原因**: 表字段命名与实际不符
- **解决方案**: 根据实际表结构调整SQL语句
