## 📚 示例条目

### 2026-03-24 | TASK_V3_101 | 回测引擎 reset() 调用时机错误

**问题类型**: 逻辑错误

**背景**: 
Gemini 验收阶段 1 时，发现回测测试结果全为 0，交易无法触发。

**根因分析**:

#### 直接原因
`api/backtest-engine.js` 的 `run()` 方法中，`this.reset()` 在获取交易日列表之前被调用，导致刚获取的 `this.dates` 被清空。

```javascript
// 错误代码（第 410 行附近）
async run() {
  this.reset();  // ❌ 这里调用会清空 dates
  this.dates = await this.getTradingDates();
  // 后续循环使用 this.dates，但已被清空
}
```

#### 深层原因
1. **代码审查不充分**: 开发完成后未经过完整自测
2. **测试覆盖不足**: 缺少端到端回测测试
3. **状态管理意识不足**: 未意识到 `reset()` 会清空已设置的状态

**解决方案**:

#### 修改文件
- `api/backtest-engine.js` - 第 410 行附近

#### 关键代码变更
```javascript
// 修复后
async run() {
  this.dates = await this.getTradingDates();
  this.reset();  // ✅ 在获取 dates 之后调用
  // 或者完全移除这里的 reset()，因为构造函数已调用
}
```

**预防措施**:

1. ✅ 关键状态变更方法（如 `reset()`）必须审查调用时机
2. ✅ 回测引擎必须进行端到端测试，验证完整流程
3. ✅ 代码审查时重点关注状态管理逻辑

### 2026-03-24 | TASK_V3_101 | 日期格式不匹配导致查询失败

**问题类型**: 配置管理

**背景**: 
回测引擎查询 `stock_daily` 表时无法获取价格数据，导致交易无法执行。

**根因分析**:

#### 直接原因
`stock_daily` 表使用 `YYYY-MM-DD` 格式，但 `getStockPrice()` 函数在查询前去掉了连字符，变成了 `YYYYMMDD`。

```javascript
// 错误代码
const dbDate = date.replace(/-/g, '');  // ❌ 去掉连字符
const sql = `SELECT * FROM stock_daily WHERE trade_date = '${dbDate}'`;
```

#### 深层原因
1. **数据字典缺失**: 未明确记录各表的日期格式
2. **假设驱动开发**: 假设所有表都使用相同格式
3. **测试数据不足**: 测试时未覆盖真实数据场景

**解决方案**:

#### 修改文件
- `api/backtest-engine.js` - `getStockPrice()` 函数

#### 关键代码变更
```javascript
// 修复后
const dbDate = date;  // ✅ 保留 YYYY-MM-DD 格式
const sql = `SELECT * FROM stock_daily WHERE trade_date = '${dbDate}'`;
```

**预防措施**:

1. ✅ 创建数据字典文档，记录各表字段格式
2. ✅ 日期处理函数统一封装，避免重复实现
3. ✅ 使用真实数据进行集成测试
