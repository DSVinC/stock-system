## 📋 版本策略

### 版本号规则

采用语义化版本号：`vX.Y.Z`
- **X**：主版本号（重大架构变更）
- **Y**：次版本号（新功能发布）
- **Z**：修订号（bug 修复）

### 发布流程

1. 完成阶段开发 → 2. 验收通过 → 3. 打 Git 标签 → 4. 创建快照 → 5. 更新本文档

### 当前版本判断

- `V3.0` 已于 2026-03-24 完成，`docs/PROJECT_PROGRESS.md` 中的 V3 任务已全部闭环
- 2026-03-26 至 2026-03-27 的 V4/V5 E2E 验收失败修复也已完成
- 当前口径应理解为：**V3 已完成，项目已进入 V4/V5 收口后的稳定迭代阶段**
- 旧的 `11/21（52%）` 口径只代表 V3 中途统计，现已废弃
