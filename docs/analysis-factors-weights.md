## 🔍 因子详细计算逻辑

### 1. 趋势因子 (Trend Factor) - 20%

**计算依据**: 均线排列、价格位置

| 子指标 | 权重 | 评分标准 |
|--------|------|----------|
| 均线多头排列 | 25% | MA5>MA10 (+0.25), MA10>MA20 (+0.25), MA20>MA60 (+0.25), 价格>MA20 (+0.25) |
| 价格在 MA20 上方 | 30% | 是 (+0.3), 否 (0) |
| 价格在 MA60 上方 | 20% | 是 (+0.2), 否 (0) |

**原始得分范围**: 0 ~ 1.0  
**映射公式**: `score = 0.6 + rawScore × 0.6`

**判断信号**:
- `maSignal`: 多头排列/空头排列/震荡
- 价格相对 MA20/MA60 位置

---

### 2. 动能因子 (Momentum Factor) - 18%

**计算依据**: MACD、RSI、MACD 柱状线

| 子指标 | 权重 | 评分标准 |
|--------|------|----------|
| MACD 信号 | 40% | 金叉 (1.0), 多头延续 (0.8), 死叉 (0.2), 空头 (0.3), 中性 (0.5) |
| RSI(14) | 35% | >70 超买 (0.7), >60 偏强 (0.9), >50 中性 (0.7), >40 中性 (0.5), >30 偏弱 (0.3), <30 超卖 (0.2) |
| MACD 柱状线 | 25% | 红柱>0.5 (1.0), 红柱≤0.5 (0.8), 绿柱<-0.5 (0.2), 绿柱≥-0.5 (0.4) |

**原始得分范围**: 0 ~ 1.0  
**映射公式**: `score = 0.6 + rawScore × 0.6`

**判断信号**:
- `macdSignal`: 金叉/死叉/多头延续/空头
- `rsiSignal`: 超买/超卖/中性

**原始得分范围**: 0 ~ 1.0

### 📰 7. 新闻因子 (News Factor) - 基本面监控体系 20%

**计算依据**: 行业新闻情感分析、公告事件、舆情监控

**应用场景**: 次日监控报告、条件单触发、行业监控

| 子指标 | 权重 | 评分标准 |
|--------|------|----------|
| 行业新闻情感 | 50% | 强利好 (+1.0), 利好 (+0.8), 中性 (0.5), 利空 (0.2), 强利空 (0) |
| 公告事件 | 30% | 财报超预期 (+1.0), 订单/合作 (+0.8), 中性公告 (0.5), 减持/解禁 (0.2), 处罚/调查 (0) |
| 舆情热度 | 20% | 热度上升 (+0.8), 热度稳定 (0.5), 热度下降 (0.2) |

**原始得分范围**: 0 ~ 1.0  
**映射公式**: `score = 0.6 + rawScore × 0.4`（新闻因子波动更大）

**情感分析算法**:
```javascript
// 正面词库
positiveWords = ['利好', '增长', '盈利', '突破', '上涨', '创新高', '支持', '订单', '合作', '重组', '中标']

// 负面词库
negativeWords = ['利空', '下跌', '亏损', '下滑', '风险', '警告', '违规', '调查', '处罚', '减持', '解禁']

// 情感评分
sentimentScore = (positiveMatches - negativeMatches) / totalMatches
// 范围：-1 (极度负面) ~ 1 (极度正面)

// 置信度
confidence = min(1.0, matchedKeywords / 5)
```

**推送条件**:
- 情感评分 > 0.6（强利好）或 < -0.4（利空）
- 24 小时内不重复推送相同新闻
- 飞书推送至主人

- 新闻数据库：`/Users/vvc/.openclaw/workspace/news_system/news.db`（26,350+ 条新闻）
- 核心监控模块：`api/industry-news-monitor.js`

## 📐 综合评分计算

### 加权得分公式

```javascript
weightedScore = 
  trend.score × 0.20 +
  momentum.score × 0.18 +
  valuation.score × 0.18 +
  capital.score × 0.16 +
  earnings.score × 0.16 +
  volatility.score × 0.12
```

### 评级映射

| 综合得分 | 报告评级 (1-5 分) | 决策建议 |
|----------|------------------|----------|
| ≥4.6 | 5.0 | 买入 |
| ≥4.2 | 4.0 ~ 5.0 | 买入 |
| ≥3.2 | 3.0 ~ 4.0 | 观望 |
| <3.2 | 1.0 ~ 3.0 | 回避 |

## 🎯 建仓区间与止损计算

### 建仓区间

```javascript
// 技术支撑区间
techLow = min(MA60, 布林下轨)
techHigh = MA20

// 估值调整
if PE < 20:  // 低估值
  valueLow = 价格 × 0.95
  valueHigh = 价格 × 1.15
elif PE > 40:  // 高估值
  valueLow = 价格 × 0.85
  valueHigh = 价格 × 1.05
else:
  valueLow = 价格 × 0.9
  valueHigh = 价格 × 1.1

// 综合建仓区间
buyLow = max(techLow, valueLow)
buyHigh = min(techHigh, valueHigh)

// 限制在±15% 范围内
buyLow = max(buyLow, 价格 × 0.85)
buyHigh = min(buyHigh, 价格 × 1.15)
```

### 止损点

```javascript
if decision == '回避':
  stopLoss = 价格 × 0.92  // 回避时止损 8%
else:
  stopLoss = min(建仓下限 × 0.95, 价格 × 0.92)
```

### 目标价

```javascript
if decision == '回避':
  targetPrice = 价格 × 1.05
else:
  // 基于评分确定倍数
  if reportScore >= 4.5:
    multiplier = 1.25
  elif reportScore >= 3.5:
    multiplier = 1.15
  else:
    multiplier = 1.08
  
  targetPrice = max(建仓上限 × multiplier, 布林上轨)
```
