## 一、正确设计标准

### 1.1 选股页面定位

**页面名称**: 行业选股推荐  
**核心功能**: 基于四维度评估模型，筛选当前市场最具投资价值的行业方向

**设计要点**:
- ✅ 展示行业方向卡片（非个股筛选器）
- ✅ 每个行业显示：综合得分、四维度评分、板块资金、热度排名、核心标的
- ✅ 支持多选行业方向
- ✅ 确认后跳转到 analysis.html 并传递选中的行业信息

### 1.2 与个股分析页的逻辑关系

```
选股页面 (select.html)
    ↓
用户选择行业方向（多选）
    ↓
点击"确定所选方向"
    ↓
跳转到 analysis.html?industry=XXX
    ↓
个股分析页自动加载该行业的成分股
    ↓
用户选择具体股票进行深度分析
```

**数据传递方式**: sessionStorage 或 URL 参数

**现象**: 用户选择行业方向后跳转到 analysis.html，但个股分析页无法正常加载该行业的成分股

2. analysis.html 未正确处理行业参数

4. 前端 JavaScript 逻辑错误

**待验证**:
- [ ] select.html 跳转时传递的数据格式
- [ ] analysis.html 接收数据的逻辑
- [ ] 成分股 API 是否存在且正常工作
