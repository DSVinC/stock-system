## 📋 讨论背景

### 当前问题
- 回测系统是**人工选择、手动迭代**，不是自动跑回测优化策略
- 与 Gemini 建议的"AI 驱动策略研究和优化"有差距
- 选股逻辑基于当前行情，但回测需要历史数据，两者脱节

### 核心困惑
> "如果现在选的股是近几年上市的新公司，回测历史数据是不是没用了？因为那时候可能还没这个公司。"

---

## 💡 核心洞察（共识）

### 用户的理解（完全正确 ✅）
```
利用回测来优化选股策略 → 使用回测出的阿尔法策略选股 → 模拟交易验证实际收益
```

### 回测的目的
| 问题 | 回答 |
|------|------|
| 回测为了什么？ | "如果我用这个选股逻辑，历史能赚多少钱？" |
| 模拟盘为了什么？ | "这个选股逻辑在实盘真的能赚钱吗？" |
| 监控为了什么？ | "实际收益和回测预期一致吗？需要调整吗？" |

## 🎯 分层设计（共识）

### Layer 1: 选股层（已有 ✅）
```
行业筛选 → 方向选股 → 个股分析 → 监控池
```
- 基于当前行情和基本面
- 输出：候选股票列表
- **改进**: 过滤掉上市时间不足的股票（如<2 年）

### Layer 2: 策略层（待建 🆕）
```
策略模板库 → 自动回测 → 参数优化 → 策略评分
```
- 策略模板：价格突破、均线交叉、估值回归等
- 自动回测：对每只候选股票自动跑多个策略
- 输出：最优策略 + 参数

### Layer 3: 执行层（部分已有 ✅）
```
策略信号 → 条件单生成 → 自动监控 → 执行交易
```
- 根据回测结果自动生成条件单
- 监控触发条件
- 执行交易并记录

## 📊 两种回测场景

### 场景 1: 策略验证型回测 ✅
```
我有一个策略规则
    ↓
用历史数据验证这个策略是否有效
    ↓
有效 → 应用到实盘
无效 → 调整策略
```

**适用对象**:
- 成熟股票（上市 3-5 年以上）
- 基于技术指标的策略（均线、RSI 等）
- 需要历史数据验证的策略参数

### 场景 2: 选股验证型回测 ⚠️ (缺失)
```
我有一个选股逻辑
    ↓
用历史数据回看：如果当时按这个逻辑选股会怎样
    ↓
验证选股逻辑的长期有效性
```

**适用对象**:
- 行业筛选逻辑
- 基本面选股标准
- 估值 + 成长因子组合

## 📋 实施路线图（共识）

### Phase 1: 数据层（1-2 周）
- [ ] 完善股票上市时间数据
- [ ] 建立策略模板库
- [ ] 回测结果持久化

### Phase 2: 自动回测（2-3 周）
- [ ] 对候选股票自动跑策略模板
- [ ] 策略评分和排序
- [ ] 参数自动优化

### Phase 3: 自动迭代（3-4 周）
- [ ] 根据回测结果自动生成条件单
- [ ] 策略效果追踪和再平衡
- [ ] AI 辅助策略研究（Gemini 建议的核心）

## 🎯 立即可做的改进（共识）

在大规模改造前，先做这些：

1. **选股时过滤新股**: 在选股逻辑中加入上市时间过滤
2. **回测结果标记**: 回测报告中标注"数据充足"或"数据不足"
3. **策略模板化**: 把常用策略做成模板，减少手动配置

## 📝 关键决策

| 决策点 | 决策内容 | 理由 |
|--------|---------|------|
| 回测定位 | 选股验证工具，不是事后验证 | 回测应指导选股，不是验证已选股票 |
| 新股处理 | 过滤掉上市<2 年的股票 | 历史数据不足，回测无意义 |
| 策略库 | 先建 3-5 个基础策略模板 | 快速验证，后续扩展 |
| 优先级 | 持仓监控 P0 → 选股回测 P1 → 策略库 P2 | 先解决紧急风控需求 |
