## 问题背景
用户发现策略版本历史页面中：
- 夏普比率 2.26、最大回撤 7.56% 等指标正常显示
- 但总收益显示为 `¥0`
- 用户质疑：有交易产生夏普比率，总收益不应该为 0

## 根因分析
1. 数据库表 `strategy_versions` 中 `total_return` 字段存储的是**收益率**（小数格式）
2. 前端 `iteration-manager.html` 显示时误用金额格式化
3. 小数收益率（如 0.402441）四舍五入到整数显示为 `¥0`

## 修复方案
将总收益显示从金额格式改为收益率百分比格式。

**修改文件**: `iteration-manager.html`

**修改位置**: 策略版本历史卡片中的评分指标显示部分

**修改内容**:
```javascript
// 修改前
['总收益', v.total_return, value => `¥${Number(value).toLocaleString(undefined, { maximumFractionDigits: 0 })}`]

// 修改后
['总收益', v.total_return, value => `${(Number(value) * 100).toFixed(2)}%`]
```

## 显示效果
| total_return | 修改前 | 修改后 | 说明 |
|-------------|--------|--------|------|
| 0.402441 | ¥0 | 40.24% | 正常收益 |
| 0.129606 | ¥0 | 12.96% | 正常收益 |
| -0.001293 | ¥0 | -0.13% | 轻微亏损 |
| 1.500000 | ¥2 | 150.00% | 高收益 |
