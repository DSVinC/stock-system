## 任务背景

股票投资系统需要盘后监控功能，每日收盘后自动检查持仓股票的技术指标变化和事件，生成监控报告并推送异常事项。

## 需求规格

### 功能需求

#### 1. 技术指标监控（盘后）
- **输入**: 持仓股票列表（从模拟账户数据库读取）
- **监控内容**:
  - 当前价格 vs 分析报告中的关键价位
    - 止损价位（硬止损/动态止损）
    - 止盈价位（第一目标/第二目标）
    - 建仓区间（是否进入建议买入区间）
  - 技术指标变化
    - MA5/MA10/MA20 趋势
    - MACD 状态
    - RSI 超买超卖
    - 成交量变化
- **输出**: 触发关注条件的股票列表

#### 2. 事件监控
- **监控内容**:
  - 个股相关新闻/公告（新浪财经）
  - 财报发布提醒（根据财报日期）
  - 关键观察点事件（来自分析报告）
- **输出**: 相关事件汇总

#### 3. 报告生成与推送
- **报告内容**:
  - 监控时间范围（昨日收盘 - 今日收盘）
  - 持仓股票状态概览
  - 触发关注条件的股票详情
  - 相关事件列表
  - 建议操作（买入/卖出/持有/关注）
- **推送方式**: 飞书私聊推送

### 技术约束

1. **运行时间**: 每日收盘后（15:30之后）
2. **运行方式**: 支持手动执行和定时任务（cron）
3. **数据源**:
   - 模拟账户数据库: `stock-system/data/simulation.db`
   - 个股分析报告: `report/stockana/YYYYMMDD/*.json`
   - 实时行情: 新浪财经
4. **技术栈**: Node.js / Python（与现有系统一致）
