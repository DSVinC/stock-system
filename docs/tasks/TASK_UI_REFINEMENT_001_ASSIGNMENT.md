# 任务分配单：TASK_UI_REFINEMENT_001

**任务名称**: 迭代管理器与策略导入体验重构（导航统一 + 版本治理 + 导入可用性）  
**优先级**: P0  
**预计工时**: 10h  
**状态**: completed  

---

## 任务背景

当前存在 3 个高频体验问题：

1. `iteration-manager.html` 风格与主站不一致，且缺少导航，用户容易“走丢”。
2. 策略自迭代管理器“只会产出版本，不会管理版本”，无效版本堆积、筛选能力不足。
3. `select.html` 从策略库导入时，版本过多难定位，且“取消/导入”按钮需滚动到底才可见。

---

## 任务范围

### A. 页面风格与导航统一（P0）

- 统一 `iteration-manager.html` 与主页面视觉风格（色板、容器、按钮层级）。
- 增加顶层导航栏（可回到 `select/backtest/analysis/monitor-pool/conditional-order`）。

### B. 策略版本管理能力补齐（P0）

- 在迭代管理器版本列表增加可用筛选：`策略类型 / 分数区间 / 更新时间 / 可发布状态`。
- 在“反馈状态暂无反馈”的阶段，降低该筛选权重，避免空筛选误导。
- 增加基础版本管理动作：`归档无效版本`（支持单条归档 + 取消归档）。

### C. 选股导入可用性优化（P0）

- `select.html` 策略导入列表增加筛选与排序（同 B 口径，先支持前端筛选）。
- 导入弹层新增固定底部操作栏（`取消 / 导入选中参数`），不受滚动位置影响。

---

## 实现要求

1. 保持现有 API 兼容，不破坏已发布流程。
2. 筛选缺省值应可回退到“全部”，不能导致空白页。
3. 无效/低分版本不删除，先走“归档”语义，保留审计可追溯。
4. 前端改造优先低风险增量，不做整页重写。

---

## 验收标准

- [x] `iteration-manager.html` 顶部导航可用，能跳转到主流程页面。
- [x] 迭代管理器版本列表可按至少 3 个条件组合筛选。
- [x] 版本列表支持归档动作（单条归档 + 取消归档）。
- [x] `select.html` 导入策略支持筛选/排序，目标策略可快速定位。
- [x] 导入弹层底部按钮固定可见，不需滚动到底部。
