### ✅ 2. 账户管理 - 清空持仓功能
- **修改文件**: 
  - `api/portfolio.js` - 添加 `clearPositions` API
  - `api/server.js` - 添加路由 `POST /api/portfolio/account/:id/clear-positions`
  - `portfolio.html` - 添加清空持仓按钮和 `clearAllPositions` 函数
- **功能说明**:
  - 在持仓概览页面添加「清空持仓」按钮
  - 清空所有持仓并按当前价格生成卖出交易记录
  - 资金退回账户初始状态
  - 解决无法删除账户的问题（先清空持仓再删除）

### ✅ 3. 账户管理 - 删除账户功能
- **修改文件**: 
  - `api/portfolio.js` - 添加 `deleteAccount` API
  - `api/server.js` - 添加路由 `DELETE /api/portfolio/account/:id`
  - `portfolio.html` - 添加删除按钮和 `deleteAccount` 函数
- **功能说明**:
  - 每个账户卡片右侧添加删除按钮（🗑️）
  - 删除前检查：无持仓、无条件单、无交易记录
  - 保护数据安全，防止误删

## 待完成任务

### ⏳ 5. 条件单 - 导入分析报告流程优化
- **问题**: 当前需要用户手动复制粘贴 JSON 报告，体验差
- **目标**: 
  1. 展示监控池股票列表（单选）
  2. 选择报告（报告编号规则：日期 + 当日次数，如 20260322002）
  3. 选择风格（激进/稳健/保守，对应报告中的建议）
  4. 自动映射条件单参数
- **涉及文件**:
  - `conditional-order.html` - 新建条件单弹窗
  - `analysis.html` - 报告生成和存储
  - `api/analysis.js` - 可能需要添加报告历史查询 API
- **优先级**: P1
- **预计工作量**: 4-6 小时

### ⏳ 6. 个股分析 - 统一决策建议和推荐评分展示
- **问题**: 方向个股列表的卡片信息与报告不一致
- **目标**:
  1. 卡片展示的决策建议和推荐评分应取自报告
  2. 报告默认每天首次查询时生成
  3. 后续需要手动点击「再次生成报告」才能生成最新报告
  4. 将报告中的「研究评级」改为「推荐评分」保持统一
- **涉及文件**:
  - `analysis.html` - 方向个股列表渲染逻辑
  - `api/analysis.js` - 报告生成和缓存逻辑
  - `api/score-factors.js` - 可能涉及研究评级字段
- **优先级**: P1
- **预计工作量**: 3-4 小时
