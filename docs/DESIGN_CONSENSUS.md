## 一、核心投资理念

### 双轮驱动策略

系统采用"中长期为主 + 短期量化为辅"的双轮驱动投资策略：

| 策略类型 | 定位 | 实现功能 |
|----------|------|----------|
| **中长期** | 主力策略 | 监控面板 + 条件单 + 模拟炒股 |
| **短期量化** | 辅助策略 | 回测系统 + 技术策略优化 |

### 目标

- 通过中长期价值投资把握核心收益
- 通过短期量化波段获取超额收益
- 实现更高的整体投资回报

---

## 二、功能模块定位

### 2.1 监控面板 + 条件单

**定位**: 中长期投资策略验证工具

**核心逻辑**:
- 基于个股分析报告的决策意见
- 结合模拟炒股验证分析报告的有效性
- 面向中长期投资周期

**决策意见来源**:
1. 基本面变化
2. 止损价位
3. 止盈价位
4. 进入建仓区间

### 2.2 回测系统

**定位**: 短线操作 + 多股票量化策略优化工具

**核心逻辑**:
- 针对短线操作策略
- 多股票组合回测
- 技术指标策略参数优化

### 2.3 模拟炒股

**定位**: 策略验证工具

**核心逻辑**:
- 验证个股分析报告意见是否科学合理
- 实盘前的策略验证沙盒

## 三、条件单重构设计

### 3.1 当前问题

当前条件单只支持纯技术面条件：
- 价格、涨跌幅、量比、RSI、MACD、PE百分位、主力净流入

**问题**: 缺乏与个股分析报告决策意见的关联

### 3.2 重构方向

条件单应基于个股分析报告的决策意见，而非纯技术指标：

| 决策类型 | 报告字段 | 条件单动作 |
|----------|----------|------------|
| 止损 | 硬止损/动态止损价位 | 股价 <= X 时卖出/提醒 |
| 止盈 | 第一目标/第二目标 | 股价 >= X 时卖出/提醒 |
| 建仓 | 建仓位置/建仓条件 | 股价进入区间时买入/提醒 |
| 加仓 | 加仓条件 | 条件满足时加仓 |
| 观察事件 | 关键观察点 | 事件发生时提醒 |
| 基本面变化 | 财报/业绩预期 | 数据更新时提醒 |

### 3.3 条件单生成时机

**采用方案C**: 用户在条件单页面选择"从分析报告导入"

**流程**:
```
条件单页面 → 点击"从分析报告导入" → 选择股票 → 读取报告JSON → 提取决策意见 → 自动生成条件单
```

### 7.2 监控频率设计

#### 每日检查（早上 9:00）

合并检查公告和估值：

| 类型 | 指标 | 数据源 |
|------|------|--------|
| **估值** | PE、PB、市值变化 | daily_basic |
| **公告** | 财报预告 | forecast |
| **公告** | 大股东减持 | stk_holdtrade |
| **公告** | 股权解禁 | share_float |

#### 收盘后检查（15:05）

检查分钟行情，判断止损/止盈/建仓触发：

| 条件 | 触发动作 | 执行价格 |
|------|----------|----------|
| 分钟最低价 ≤ 止损价 | 卖出 | 触发时的价格 |
| 分钟最高价 ≥ 止盈价 | 卖出 | 触发时的价格 |
| 分钟价格进入建仓区间 | 建仓 | 触发时的价格 |

### 7.3 按时间顺序模拟交易

收盘后检查分钟行情时，**按时间顺序**处理，模拟真实行情走势：

| 时间 | 价格 | 触发动作 |
|------|------|----------|
| 10:30 | 10.80 | 进入建仓区间 → 建仓 5% |
| 14:00 | 9.70 | 触及止损 → 卖出 |
| 14:30 | 10.20 | （已止损，不再处理） |

## 八、策略结构化设计（2026-03-19 新增）

### 8.1 设计原则

策略必须配合监控服务，需要机器可读的结构化数据。

### 8.2 结构定义

在分析报告 JSON 中增加 `strategy` 字段：

```json
{
  "buyZone": [10.50, 11.20],
  "stopLoss": 9.80,
  "targetPrice": 13.50,
  "strategy": {
    "firstBuyPercent": 0.05,
    "maxPosition": 0.15,
```

### 8.3 字段说明

| 字段 | 类型 | 说明 |

| `addOnConditions` | array | 加仓条件列表 |
| `riskProfile` | string | 风险偏好（aggressive/balanced/conservative） |

## 九、条件单与模拟账户联动逻辑（2026-03-19 新增）

### 9.1 触发流程

条件单被触发时，执行以下流程：

```
条件单触发
    │
    ├─→ 飞书推送（通知主人）
    │
    └─→ 模拟账户自动执行（买入/卖出）
        │
        ├─→ 记录交易明细
        ├─→ 计算交易成本（手续费、税费）
        ├─→ 更新持仓
        └─→ 计算持仓盈亏
```

**关键原则**：推送和模拟操作**同步进行**，不需要主人确认后才执行。

### 9.2 模拟账户功能

| 功能 | 说明 |
|------|------|
| **交易明细记录** | 每次买入/卖出操作都完整记录 |
| **成本计算** | 计算A股相关手续费、税费、分红 |
| **持仓管理** | 实时更新持仓数量和成本价 |
| **盈亏追踪** | 根据持仓情况实时计算浮动盈亏 |
| **数据存储** | 所有数据存入数据库 |

### 9.3 数据模型

#### 账户表 (simulation_account)

| 字段 | 类型 | 说明 |
|------|------|------|
| `account_id` | string | 账户ID |
| `initial_capital` | decimal | 初始资金（用户输入，≤1,000,000的整数） |
| `available_cash` | decimal | 可用资金 |
| `total_market_value` | decimal | 持仓市值 |
| `total_assets` | decimal | 总资产 |
| `created_at` | datetime | 创建时间 |

**初始资金配置规则**：
- 用户在前端界面自行输入
- 必须是不大于100万的整数（1~1,000,000）
- 输入后不可修改（除非清空账户重新开始）

#### 交易记录表 (simulation_trade)

| 字段 | 类型 | 说明 |
|------|------|------|
| `trade_id` | string | 交易ID |
| `account_id` | string | 账户ID |
| `stock_code` | string | 股票代码 |
| `trade_type` | enum | buy/sell |
| `trade_price` | decimal | 成交价格 |
| `trade_volume` | int | 成交数量 |
| `trade_amount` | decimal | 成交金额 |
| `commission` | decimal | 手续费 |
| `tax` | decimal | 税费 |
| `total_cost` | decimal | 总成本 |
| `trade_time` | datetime | 交易时间 |
| `triggered_by` | string | 触发来源（条件单ID） |

#### 持仓表 (simulation_position)

| 字段 | 类型 | 说明 |
|------|------|------|
| `position_id` | string | 持仓ID |
| `account_id` | string | 账户ID |
| `stock_code` | string | 股票代码 |
| `stock_name` | string | 股票名称 |
| `hold_volume` | int | 持仓数量 |
| `avg_cost` | decimal | 平均成本价 |
| `current_price` | decimal | 当前价格 |
| `market_value` | decimal | 持仓市值 |
| `floating_pnl` | decimal | 浮动盈亏 |
| `floating_pnl_pct` | decimal | 浮动盈亏率 |

### 9.4 成本计算规则

| 费用类型 | 计算方式 |
|----------|----------|
| **佣金** | 成交金额 × 0.025%（最低5元） |
| **印花税** | 卖出金额 × 0.1%（买入不收） |
| **过户费** | 成交金额 × 0.002%（沪市） |
| **分红税** | 按持股时间分档（1月内20%，1年-1月10%，1年以上0%） |

### 9.5 盈亏计算频率

| 计算方式 | 触发时机 | 数据来源 |
|----------|----------|----------|
| **自动计算** | 每日收盘后（15:05） | 当日收盘价 |
| **手动刷新** | 用户主动触发 | 实时行情 |

**限制规则**：
- 手动刷新每日限 **3次**
- 超出次数提示「今日刷新次数已用完，请明日再试」

## 十、行业新闻监控设计（2026-03-19 新增）

### 10.1 核心逻辑

行业新闻作为**基本面因子**融入监控系统，与个股分析报告中的行业信息联动。

```
个股分析报告
    │
    ├─→ 明确归属行业（行业代码）
    ├─→ 提取行业关键词
    │
    └─→ 进入监控池
            │
            └─→ 每小时抓取该行业新闻
                    │
                    ├─→ 关键词匹配
                    ├─→ AI判断利好/利空
                    │
                    ├─→ 存入「行业新闻因子」表（次日报告使用）
                    └─→ 飞书实时推送（立即通知）
```

### 10.2 新闻抓取与分析流程

| 步骤 | 动作 | 输出 |
|------|------|------|
| **抓取** | 每小时从新闻数据库抓取 | 原始新闻列表 |
| **匹配** | 用行业关键词筛选相关新闻 | 匹配新闻列表 |
| **AI判断** | 分析利好/利空/中性 | 情感标签 + 置信度 |
| **存储** | 存入行业新闻因子表 | 结构化数据 |
| **推送** | 飞书通知 + 传入监控系统 | 实时提醒 + 因子数据 |

### 10.3 基本面因子权重设计（初版）

| 因子类别 | 权重 | 说明 |
|----------|------|------|
| **估值变化** | 25% | PE/PB/PS 偏离度 |
| **财务指标** | 25% | ROE/毛利率/营收同比等 |
| **行业新闻** | 20% | 利好/利空综合评分 |
| **公告事件** | 20% | 财报/减持/解禁等 |
| **资金流向** | 10% | 主力/北向资金变化 |

**权重优化原则**：
- 权重作为**可调节参数**存入系统（具体位置待定）
- 根据**模拟交易结果**持续优化
- 这是系统的**核心能力**之一

### 10.4 数据模型

#### 行业新闻因子表 (industry_news_factor)

| 字段 | 类型 | 说明 |
|------|------|------|
| `news_id` | string | 新闻唯一ID |
| `industry_code` | string | 行业代码 |
| `industry_name` | string | 行业名称 |
| `news_title` | string | 新闻标题 |
| `news_url` | string | 新闻链接 |
| `publish_time` | datetime | 发布时间 |
| `sentiment` | enum | 情感判断：strong_positive/positive/neutral/negative/strong_negative |
| `confidence` | decimal | 置信度（0-1）|
| `keywords_matched` | array | 匹配的关键词列表 |
| `is_notified` | boolean | 是否已飞书推送 |
| `created_at` | datetime | 入库时间 |

**去重机制**：按新闻URL/标题哈希去重，24小时内不重复推送。

### 10.5 监控报告整合

次日 9:00 生成监控报告时：

```
读取所有基本面因子
    │
    ├─→ 估值变化评分 × 25%
    ├─→ 财务指标评分 × 25%
    ├─→ 行业新闻评分 × 20%（24小时内所有新闻综合）
    ├─→ 公告事件评分 × 20%
    └─→ 资金流向评分 × 10%
            │
            └─→ 加权计算综合评分
                    │
                    └─→ 给出监控意见 → 飞书推送
```

## 十一、回测系统设计（2026-03-19 新增）

### 11.1 方案选择

**采用方案C：混合模式（事件驱动 + 向量化）**

### 11.2 两阶段流程

```
┌─────────────────────────────────────────────────────────────┐
│  第一阶段：向量化批量筛选（粗筛）                              │
│  ─────────────────────────────────                           │
│  输入：100+ 策略参数组合                                       │
│  处理：批量计算信号，快速统计收益                              │
│  输出：胜率、盈亏比、年化收益排名                              │
│  目标：从大量策略中快速筛选Top 10                            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  第二阶段：事件驱动精细验证（精测）                            │
│  ─────────────────────────────────                           │
│  输入：Top 10 策略                                            │
│  处理：模拟真实交易流程，计算滑点、成本                        │
│  输出：详细交易明细、收益率曲线、最大回撤、夏普比率            │
│  目标：精细验证策略真实表现                                    │
└─────────────────────────────────────────────────────────────┘
```

### 11.3 各阶段详细设计

#### 第一阶段：向量化批量筛选

| 配置项 | 说明 |
|--------|------|
| **数据粒度** | 日级或分钟级行情 |
| **策略参数** | 技术指标组合（MA/RSI/MACD/布林带等）|
| **计算方式** | 向量化批量计算，不模拟逐笔交易 |
| **评估指标** | 胜率、盈亏比、年化收益率、交易次数 |
| **筛选标准** | 综合评分排名Top 10 |

#### 第二阶段：事件驱动精细验证

| 配置项 | 说明 |
|--------|------|
| **数据粒度** | 分钟级历史行情 |
| **触发机制** | 价格触及策略阈值时模拟成交 |
| **执行逻辑** | 按时间顺序处理，模拟滑点和延迟 |
| **成本计算** | 同模拟账户（佣金、印花税、过户费）|
| **评估指标** | 总收益率、年化收益、最大回撤、夏普比率、Calmar比率 |

### 11.4 系统关系说明

**回测系统** 与 **监控系统** 是两个独立功能：

```
┌──────────────────────────────────────────────────────┐
│              回测系统（独立功能）                       │
│  ────────────────────────────────────────────────   │
│  用途：短期量化策略优化                                 │
│  数据：历史行情                                         │
│  目标：找最优参数组合                                   │
│  输出：策略排名报告                                     │
└──────────────────────────────────────────────────────┘
                    ↓ 策略有效时
                    （可导入）
┌──────────────────────────────────────────────────────┐
│              监控系统（独立功能）                       │
│  ────────────────────────────────────────────────   │
│  用途：中长期策略验证                                   │
│  数据：实时行情                                         │
│  目标：执行既定策略                                     │
│  输出：实时交易信号                                     │
│                                                        │
│  包含子系统：                                           │
│  ├─→ 模拟账户（含条件单触发、自动执行、成本计算）        │
│  ├─→ 基本面监控（估值、财务、行业新闻、公告）            │
│  └─→ 行业新闻监控（每小时抓取→AI判断→双重推送）          │
└──────────────────────────────────────────────────────┘
```

**关键原则**：
1. 回测系统与监控系统独立运行
2. 模拟账户是**监控系统的一部分**，与回测系统无关
3. 回测验证有效的策略可导入监控系统执行
